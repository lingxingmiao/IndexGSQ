# IndexGSQ

Language: [中文](https://github.com/lingxingmiao/IndexGSQ/blob/main/README-ZH.md)/English

1GB of embedding vectors can be indexed using ~90MB of memory, while maintaining **92%+ recall** at 2.5 bits (on 383,113 vectors).

### Recall Rates
- 8‑bit: up to 100%
- 6‑bit: up to 99.9%
- 4‑bit: up to 99.5%
- 3‑bit: up to 98.4%
- 2‑bit: up to 93.9%

All code is generated by AI.

### Core Concepts
- **Vector Re‑ordering** – First, a coarse search is performed using Faiss IVF, followed by a fine‑grained cosine similarity computation for each block to select the top‑k‑1 vectors. In total, each block contains k vectors.
- **GSQ_K** – Rotates and scales the block directions to reduce the variance across different dimensions within the same block.

### Major Limitations
- Cannot dynamically add vectors (due to vector re‑ordering and GSQ_K).
- Text metadata must be re‑ordered synchronously.
- High memory consumption during index building (about twice the size of the input vectors).
- Relatively slow indexing speed (~2.5 vectors per second on my machine).

The core content and recall data are sourced from [TranslatorMinecraft](https://github.com/lingxingmiao/Translator-Minecraft/) version R1.6 B2.

### Usage

```python
import indexgsq
import numpy as np

np.random.seed(42)  # for reproducible tests

# Create the index (default parameters are shown, they can be omitted)
index = indexgsq.IndexGSQKCosine(
    vectors_block=128,      # number of vectors per sub‑block for quantization
    reranker_block=128,     # size of each cluster block during re‑ordering
    quantization=2          # quantization bits: 2, 3, 4, 6, or 8
)

# Generate some random vectors (e.g., 10,000 vectors of dimension 1024 → ~312.5 MB)
vectors = np.random.randn(10000, 1024).astype(np.float32)
index.add(vectors)          # this step is time‑consuming and memory‑hungry

# Query
query = np.random.randn(1024).astype(np.float32)
scores, ids = index.search(query, k=10)   # retrieve top‑10
print("Top‑10 indices:", ids[0])

# Associate text metadata
texts = [f"text_{i}" for i in range(10000)]   # dummy text labels
reordered_texts = index.text(texts)            # re‑order texts to match internal ordering
top_texts = [reordered_texts[i] for i in ids[0] if i != -1]
print("Top‑10 texts:", top_texts)

# Optionally save the re‑ordered texts together with the index
index.save_text(texts)          # automatically re‑orders and stores

# Save the full index (extension is arbitrary)
index.save("index.gsqk")        # the saved file size for the above vectors is theoretically ~2.77 MB

# Load the index later
new_index = indexgsq.load("index.gsqk")
new_texts = new_index.texts     # the previously saved texts are available
```
